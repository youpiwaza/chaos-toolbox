Chaos toolbox / Nude html

A big but simple bulk of all html tags you usually use in website.
The goal here is to allow you to verify your styles, and also the overall rendering & harmony.
It can also helps to prevent bug and styles conflicts (espceially on updates, it'll help prevent regressions).
And finally this will provide a solid base to create html, providing best practice, optimisations, compatibility and accessibility.

Recommandations
  • Style / No matter what, those tags should remain inline
  • Style / When styling your links, you should have 3 colors :
    • One for the initial state
      a, a:active { ... }
    • One for the visited state
      a:visited { ... }
    • One for the user interaction
      a:focus, a:hover { ... }
    • Pay attention to declare those in this specific order, as CSS priority is needed (ex : hover style should display over visited).
      Care if you are used to alphabetical sorting.
  • Compatibility / Even if both the b & i tags are deprecated, you should have basic style for them
  • SEO / You should use strong & em tags to emphasize your contents. It makes reading easier, and also helps robots getting keywords
  • SEO / Links > Always give a title to your links. Preferably diffrent from the title label
  • SEO / Links > Don't forget to use the target attribute. If the link is targeting a resource outside your website, you should go for a target _blank
    If you got a lot of mixed links (internal & external), I'd recommand to add a specific inline icon.

Simple

Verify behavior if an image is too big. It shouldn't overflow it's parent container.

Responsive image (recommanded)

Provide different images (low size for mobile, average for tablet, high quality for desktop), allwing for data comsumption savings.
If you want to test with Chrome Dev Tools, make sure to check "Disable cache" in the network tab.

Always consider what are your responsive goals (supports, breakpoints) before going in.

Note that you can make the image creation process pretty quick n' easy thanks to Photoshop scripts
No needs for plugins or 3rd parties, just record a script where you resize/export for web one image to desired sizes, then use File > automation > batch.

  • Take care to limit images max-width to prevent design explosions, here's a simple fix
    img {

}
  • Take care for images copyrights if you get resources from the web. Prefer going for your own images or images stocks (adobe, shutterstock, gettyimages, etc.)
  • Same, prefer hosting your images (and not using external links as img src attributes to prevent 404
  • SEO & compatibility / Prefer using responsive images
  • SEO & Compatibility / Take care to optimize your images (svg if possible), I'd recommand jpg, progressive, quality 60 ; they are some automated stuff online/through npm/gulp
  • SEO / Always specify title & alt attributes. Learn the difference.
  • SEO / Always specify natural images height & width

  • Accessibility / Decorative images, set alt attribute empty, and use role="presentation" instead

Form & form elements

<form> tag

If classic form (not js handled), consider the following attributes :
  • method : get/post
  • action : target page
  • target : (self / new tab), usually not used
Be careful, those attributes can be overided with button or input (usually type button or image) following attribute
  • formaction
  • formmethod
  • formtarget
If using javascript (or other) method to handle form submission, pay attention to
  • Still use the submit event, in addition with preventDefault js function
    • The reason for that is that users can send their form in other ways than just pushing the submit button, like
      • Pressing the "Enter" key while a field is focused
      • Using tabulations to switch focus to the submit button, then pressing space
      • Other mobile/tablet/accessibility readers shenanigans
  • Pay attention to attach the submit event the form, and not to the submit button
  • Consider using an input type button instead of submit, if you wanna skip preventDefault()

Layout recommandations

  • All fields should be displayed with a <label> tag associated. Pay attention po properly fill the for attribute (label) & the corresponding id attribute (~input)
  • Consider regrouping fields by thematics. Some websites recommand using <section> tags, associated with optionnal title tags.
  • Make use of <fieldset> & <legend>, which provides a contour and a title
  • In case of a very tall form, consider using the new sticky css stuff : Air mozilla, sticky position presentation, and here's a demo.
    Could be especially usefull on mobile/tablet to keep buttons always available to users.
  • Ergonomy, jwegesin presentation "Forms sucks". Recap :
    • Prefer vertical stacking
    • Add labels on top of fields
    • Buttons
      • Add colors (green for validation, red to cancel)
      • If different behavior, redirections (like a lost password action), use links instead of buttons
      • Prioritize : left to right
      • Validation button should be vertically aligned with other fields
      • Regroup buttons
    • Don't use acronyms or programmaing oriented mambo jambo in labels or placeholders
    • Make proper use of labels & placeholders, those are differents (see below, placeholders)
    • Add only mandatory fields, especially on "first" forms (ex: online shops)
    • Add flexibility on controversial types fields, ex: phone
      It's also a great opportunity to use a clean placeholder with real data
    • Validate on the go, not only once on submit
    • Autocompletion optimisation (see below)
    • Error management : By field with proper indications, plus a conscise recap at the end on validation

Placeholder

Text illustration of what an input should be filled with.

Note: Do not use the placeholder attribute instead of a <label> element, their purposes are different. The <label> attribute describes the role of the form element (i.e. it indicates what kind of information is expected), and the placeholder attribute is a hint about the format that the content should take. There are cases in which the placeholder attribute is never displayed to the user, so the form must be understandable without it.

Make it cross browser, as it's HTML5 only, even if mostly supported.
Basically use modernizr & jquery to accomodate for older browsers

MDN doc : placeholder

readonly

Seriously, never do this
Used to display info to the user only, he can't edit it.
The value can still be edited by a script though.
Be careful, it can also be edited by editing html code in the browser through inspector !
Doesn't not work on several input types : hidden, image, or a button type (submit, reset, or button).

required

  • This attribute specifies that the user must fill in a value before submitting a form.
  • Be careful, the form won't submit until the field id populated
  • Doesn't not work on several input types : hidden, image, or a button type (submit, reset, or button).
  • CSS : The :optional and :required CSS pseudo-classes will be applied to the field as appropriate.
